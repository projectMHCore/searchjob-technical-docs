Детальні кроки тестування

№ Steps Description Expected Actual State
1
 Відкрити форму Перейти на register.php Форма відображається з полями: логін, email, пароль, підтвердження, роль
2
 Ввести логін Ввести "testuser123" Поле заповнене, без помилок
3
 Ввести email Ввести "test@example.com" Email формат коректний
4
 Ввести пароль Ввести "MySecurePass123!" Пароль прихований символами •
5
 Підтвердити пароль Ввести "MySecurePass123!" Паролі співпадають
6
 Відправити форму Натиснути "Зареєструватися" Успішна реєстрація, перенаправлення на success
7
 Тест неспівпадіння Заповнити форму знову Форма готова для негативного тесту
8
 Ввести пароль Ввести "Password123" Поле заповнене
9
 Неспівпадаючий пароль Ввести "Password124" Значення відрізняється
10
 Відправити форму Натиснути "Зареєструватися" Помилка: "Пароли не совпадают!"

Технічні деталі валідації

Код валідації з RegisterController.php: // Перевірка заповнення всіх полів if (!$login || !$password || !$email) { $error = 'Заполните все обязательные поля!'; } // Перевірка співпадіння паролів elseif ($password !== $confirm_password) { $error = 'Пароли не совпадают!'; } Обробка помилок: ob_end_clean(); header('Location: register.php?error=' . urlencode($error)); exit;

Тестові дані

✅ Валідні дані

  • Логін: testuser123, user2024
  • Email: test@example.com, user@company.org
  • Пароль: MySecurePass123!, TestPassword2024

❌ Невалідні дані

  • Неспівпадання: "Password123" / "Password124"
  • Порожні поля: "" / "somepassword"
  • Спецсимволи: "Pass<script>" / "Pass<script>"

Очікувані результати

✅ Позитивний сценарій

  • Форма відправляється успішно
  • Користувач створений в БД
  • Перенаправлення на register.php?success=registered
  • Повідомлення про успіх

❌ Негативний сценарій

  • Валідація спрацьовує на сервері
  • Помилка "Пароли не совпадают!"
  • Перенаправлення з параметром error
  • Форма з повідомленням про помилку

Postcondition

При успішному тесті: Новий користувач створений, пароль захешований, можливість авторизації

При помилці: Користувач НЕ створений, інформативна помилка, можливість повтору

Очищення: Видалити тестові дані, очистити кеш, перевірити логи

Критерії проходження
